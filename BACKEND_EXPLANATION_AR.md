# 🔧 شرح النظام الخلفي (Backend) - بالعربي البسيط

## 📚 الجزء الأول: كيف يعمل النظام الخلفي؟

### 1️⃣ ما هو الـ API؟

**API** يعني "واجهة برمجة التطبيقات" - لكن ببساطة:
- تخيل API زي **نادل في مطعم**
- أنت (الموقع) تطلب من النادل (API) تجيب لك أكل (بيانات)
- النادل يروح للمطبخ (قاعدة البيانات) ويجيب لك الطلب
- ويرجع لك بالأكل (البيانات) على طبق (JSON)

**مثال عملي:**
```
أنت تضغط على زر "عرض المنتجات"
    ↓
الموقع يطلب من API: "عايز كل المنتجات"
    ↓
API يروح لقاعدة البيانات ويجيب المنتجات
    ↓
API يرجع لك قائمة المنتجات
    ↓
الموقع يعرض لك المنتجات في جدول
```

### 2️⃣ ما هو JWT (المصادقة)؟

**JWT** يعني "JSON Web Token" - لكن ببساطة:
- تخيل JWT زي **بطاقة دخول للنادي**
- لما تسجل دخول بإيميل وباسورد صح، النظام يديك "بطاقة دخول"
- كل مرة تطلب حاجة، لازم تورّي البطاقة دي
- لو البطاقة صح → تدخل
- لو البطاقة غلط أو مش موجودة → ممنوع الدخول

**كيف يعمل؟**
```
1. تكتب إيميل وباسورد في صفحة تسجيل الدخول
2. النظام يتأكد: الإيميل ده موجود؟ الباسورد صح؟
3. لو صح → النظام يديك "توكن" (بطاقة دخول)
4. التوكن ده يتحفظ في جهازك
5. كل مرة تطلب حاجة، التوكن يروح مع الطلب
6. النظام يشوف التوكن ويقول: "أوك، أنت مسموح ليك"
```

**مثال التوكن:**
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOiIxMjM0In0.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
```
(دي مش كلام عشوائي - دي بطاقة دخول مشفرة!)

### 3️⃣ ما هي قاعدة البيانات (Prisma)؟

**Prisma** هي أداة للتعامل مع قاعدة البيانات - لكن ببساطة:
- تخيل قاعدة البيانات زي **خزانة ملفات ضخمة**
- كل درج فيها = جدول (مثلاً: درج المنتجات، درج العملاء)
- كل ورقة في الدرج = سجل (مثلاً: منتج واحد، عميل واحد)
- Prisma هي الشخص اللي بيفتح الدرج ويجيب لك الورقة اللي عايزها

**مثال:**
```javascript
// عايز كل المنتجات من قاعدة البيانات
const products = await prisma.product.findMany();

// عايز أضيف منتج جديد
const newProduct = await prisma.product.create({
  data: {
    name: "أنابيب بلاستيك 50مم",
    price: 45,
    stock: 100
  }
});
```

## 🔗 الجزء الثاني: كيف يتصل الموقع بالنظام الخلفي؟

### 1️⃣ طلبات الـ Fetch

**Fetch** يعني "إحضار" - وهي الطريقة اللي الموقع يطلب بيها بيانات:

```javascript
// الموقع يطلب بيانات من API
const response = await fetch('/api/products', {
  method: 'GET',  // نوع الطلب (GET = جيب، POST = أضف)
  headers: {
    'Authorization': 'Bearer توكن-الدخول-هنا',  // بطاقة الدخول
    'Content-Type': 'application/json'
  }
});

const products = await response.json();  // تحويل الرد لبيانات نقدر نستخدمها
```

### 2️⃣ الـ Headers (رؤوس الطلب)

**Headers** زي **مظروف الجواب** - فيه معلومات مهمة:
- `Authorization`: بطاقة الدخول (التوكن)
- `Content-Type`: نوع البيانات (JSON عادةً)

**مثال:**
```javascript
headers: {
  'Authorization': 'Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...',
  'Content-Type': 'application/json'
}
```

### 3️⃣ استخدام التوكن

**التوكن يتحفظ في localStorage** (مخزن محلي في المتصفح):

```javascript
// لما تسجل دخول، احفظ التوكن
localStorage.setItem('token', 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...');

// لما تطلب بيانات، استخدم التوكن
const token = localStorage.getItem('token');
fetch('/api/products', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

## 🔄 الجزء الثالث: كيف تسير البيانات في النظام؟

### مثال كامل: إنشاء فاتورة شراء

دعنا نتتبع رحلة البيانات من البداية للنهاية:

#### **الخطوة 1: المستخدم يملأ النموذج**
```
المستخدم يفتح صفحة "فاتورة شراء جديدة"
    ↓
يختار المورد: "شركة البلاستيك المصرية"
    ↓
يضيف منتج: "خامات PVC - 100 كيلو - سعر 12 جنيه"
    ↓
يضغط زر "حفظ"
```

#### **الخطوة 2: الموقع يرسل البيانات للـ API**
```javascript
// الموقع يجهز البيانات
const invoiceData = {
  supplierId: "abc123",           // رقم المورد
  invoiceNumber: "PO-2026-0001",  // رقم الفاتورة
  date: "2026-04-13",             // التاريخ
  items: [
    {
      productId: "xyz789",         // رقم المنتج
      quantity: 100,               // الكمية
      price: 12                    // السعر
    }
  ]
};

// الموقع يرسل للـ API
const response = await fetch('/api/purchase-invoices', {
  method: 'POST',                  // POST = إضافة جديد
  headers: {
    'Authorization': 'Bearer توكن-المستخدم',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(invoiceData)  // تحويل البيانات لنص
});
```

#### **الخطوة 3: الـ API يستقبل الطلب**
```javascript
// في ملف: app/api/purchase-invoices/route.ts

export async function POST(request) {
  // 1. التأكد من التوكن (المصادقة)
  const user = await getAuthenticatedUser(request);
  if (!user) {
    return { error: "لازم تسجل دخول الأول!" };
  }
  
  // 2. التأكد من الصلاحيات
  if (!user.permissions.includes('create_purchase_invoice')) {
    return { error: "مش مسموح لك تعمل فاتورة شراء!" };
  }
  
  // 3. استلام البيانات
  const data = await request.json();
  
  // 4. حساب الإجمالي (100 × 12 = 1200 جنيه)
  const total = data.items.reduce((sum, item) => 
    sum + (item.quantity * item.price), 0
  );
  
  // 5. حفظ في قاعدة البيانات
  const invoice = await prisma.purchaseInvoice.create({
    data: {
      supplierId: data.supplierId,
      invoiceNumber: data.invoiceNumber,
      date: new Date(data.date),
      total: total,  // 1200 جنيه
      items: {
        create: data.items.map(item => ({
          productId: item.productId,
          quantity: item.quantity,
          price: item.price,
          total: item.quantity * item.price
        }))
      }
    }
  });
  
  // 6. تحديث المخزون (زيادة الكمية)
  await updateStock(data.items, 'increase');
  
  // 7. إنشاء قيد محاسبي
  await createAccountingEntry({
    type: 'purchase',
    amount: total,
    invoiceId: invoice.id
  });
  
  // 8. إرجاع النتيجة للموقع
  return { success: true, data: invoice };
}
```

#### **الخطوة 4: قاعدة البيانات تحفظ البيانات**
```
قاعدة البيانات تفتح جدول "PurchaseInvoice"
    ↓
تضيف سجل جديد:
  - رقم الفاتورة: PO-2026-0001
  - المورد: شركة البلاستيك المصرية
  - التاريخ: 2026-04-13
  - الإجمالي: 1200 جنيه
    ↓
تفتح جدول "PurchaseInvoiceItem"
    ↓
تضيف الأصناف:
  - خامات PVC - 100 كيلو - 12 جنيه
    ↓
تفتح جدول "Product"
    ↓
تحدث المخزون: كان 50 → أصبح 150 كيلو
    ↓
تفتح جدول "JournalEntry"
    ↓
تضيف قيد محاسبي:
  - من حـ/ المشتريات (مدين 1200)
  - إلى حـ/ الموردين (دائن 1200)
```

#### **الخطوة 5: النظام المحاسبي يسجل القيد**
```javascript
// إنشاء قيد محاسبي تلقائي
const journalEntry = await prisma.journalEntry.create({
  data: {
    entryNumber: "JE-2026-0001",
    date: new Date(),
    description: "فاتورة شراء رقم PO-2026-0001",
    lines: {
      create: [
        {
          accountCode: "5100",        // حساب المشتريات
          accountName: "مشتريات خامات",
          debit: 1200,                // مدين
          credit: 0
        },
        {
          accountCode: "2100",        // حساب الموردين
          accountName: "الموردين",
          debit: 0,
          credit: 1200                // دائن
        }
      ]
    }
  }
});
```

#### **الخطوة 6: الـ API يرجع النتيجة للموقع**
```javascript
// الرد من API
{
  "success": true,
  "data": {
    "id": "invoice-123",
    "invoiceNumber": "PO-2026-0001",
    "supplier": {
      "name": "شركة البلاستيك المصرية"
    },
    "total": 1200,
    "items": [
      {
        "product": "خامات PVC",
        "quantity": 100,
        "price": 12,
        "total": 1200
      }
    ]
  },
  "message": "تم إنشاء فاتورة الشراء بنجاح"
}
```

#### **الخطوة 7: الموقع يحدث الواجهة**
```javascript
// الموقع يستقبل الرد
const result = await response.json();

if (result.success) {
  // عرض رسالة نجاح
  alert("تم إنشاء فاتورة الشراء بنجاح!");
  
  // تحديث قائمة الفواتير
  fetchInvoices();  // جلب الفواتير من جديد
  
  // إغلاق النموذج
  closeForm();
  
  // الرجوع لصفحة الفواتير
  router.push('/dashboard/purchases/invoices');
}
```

## 📊 ملخص تدفق البيانات

```
┌─────────────────┐
│  المستخدم       │ يملأ النموذج ويضغط "حفظ"
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  الموقع (React) │ fetch('/api/purchase-invoices', {...})
└────────┬────────┘
         │ يرسل: البيانات + التوكن
         ↓
┌─────────────────┐
│  API Route      │ يتحقق من التوكن والصلاحيات
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  Prisma         │ يحفظ في قاعدة البيانات
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  قاعدة البيانات │ تحفظ الفاتورة + تحدث المخزون + تسجل القيد
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  API Route      │ يرجع النتيجة
└────────┬────────┘
         │ يرجع: { success: true, data: {...} }
         ↓
┌─────────────────┐
│  الموقع (React) │ يعرض رسالة نجاح ويحدث الواجهة
└─────────────────┘
         │
         ↓
┌─────────────────┐
│  المستخدم       │ يشوف الفاتورة الجديدة في القائمة
└─────────────────┘
```

## 🔐 أمان النظام

### كيف يحمي النظام بياناتك؟

1. **تشفير الباسورد:**
   - الباسورد لا يُحفظ كما هو
   - يُحوّل لكود مشفر (hash) باستخدام bcrypt
   - حتى لو سُرقت قاعدة البيانات، الباسورد مش هيبان

2. **التوكن (JWT):**
   - التوكن له صلاحية (عادةً 7 أيام)
   - بعد كده لازم تسجل دخول تاني
   - التوكن مشفر ومحمي

3. **الصلاحيات:**
   - كل مستخدم له صلاحيات محددة
   - مثلاً: موظف المبيعات يقدر يعمل فاتورة بيع بس
   - المدير يقدر يشوف التقارير المالية
   - المحاسب يقدر يعدل القيود

4. **التحقق من كل طلب:**
   - كل طلب للـ API يتم التحقق منه
   - لو التوكن مش صح → رفض الطلب
   - لو المستخدم مش عنده صلاحية → رفض الطلب

---

## 💡 نصائح مهمة

1. **لا تشارك التوكن مع أحد** - ده زي باسوردك
2. **سجل خروج لما تخلص شغل** - خصوصاً على جهاز مشترك
3. **استخدم باسورد قوي** - أرقام + حروف + رموز
4. **راجع الصلاحيات** - كل مستخدم ياخد صلاحيات شغله بس

---

هذا شرح مبسط لكيفية عمل النظام الخلفي. النظام معقد من الداخل، لكن الفكرة الأساسية بسيطة: **الموقع يطلب → API يعالج → قاعدة البيانات تحفظ → API يرجع النتيجة → الموقع يعرض**.
