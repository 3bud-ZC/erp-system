# 📚 Prisma Database Seeding

## 🗂️ ملفات Seed المتوفرة

### 1. **seed.ts** (الافتراضي - قاعدة بيانات نظيفة)
يقوم بحذف جميع البيانات من قاعدة البيانات ويتركها فارغة ونظيفة.

**الاستخدام:**
```bash
npm run seed
# أو
npx prisma db seed
```

**ما يفعله:**
- ✅ حذف جميع الفواتير والأوامر
- ✅ حذف جميع المنتجات والعملاء والموردين
- ✅ حذف جميع البيانات الأساسية
- ✅ حذف بيانات المحاسبة
- ⚠️ **يحتفظ ببيانات المستخدمين (Users) فقط**

---

### 2. **seed-clean.ts** (تنظيف شامل)
نفس وظيفة `seed.ts` - يحذف جميع البيانات ويترك قاعدة البيانات نظيفة.

**الاستخدام:**
```bash
npx tsx prisma/seed-clean.ts
```

---

### 3. **seed-auth.ts** (بيانات المصادقة فقط)
يضيف مستخدم Admin افتراضي للنظام.

**الاستخدام:**
```bash
npx tsx prisma/seed-auth.ts
```

**بيانات الدخول الافتراضية:**
- Username: `admin`
- Password: `admin123`

---

## 🚀 الاستخدام في Production

عند نشر النظام على الإنترنت، سيتم تشغيل `seed.ts` تلقائياً والذي سيقوم بـ:
1. حذف جميع البيانات التجريبية
2. ترك قاعدة البيانات نظيفة وجاهزة للاستخدام
3. الاحتفاظ ببيانات المستخدمين فقط

---

## 📝 ملاحظات مهمة

- ⚠️ **تحذير:** تشغيل seed سيحذف جميع البيانات الموجودة!
- ✅ بيانات المستخدمين (Users) محمية ولن يتم حذفها
- 🔐 يمكنك إضافة مستخدم Admin باستخدام `seed-auth.ts`
- 🧹 النظام الآن جاهز للإنتاج بدون أي بيانات تجريبية

---

## 🛠️ للمطورين

إذا أردت إضافة بيانات تجريبية للتطوير:
1. انسخ محتوى `seed.ts` القديم من Git history
2. أو أضف البيانات يدوياً من واجهة النظام
